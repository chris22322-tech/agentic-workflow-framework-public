# SWE Learning Pathway

A 5-lesson interactive learning pathway for Software Engineers. Learn how to take engineering workflows and turn them into a platform-neutral agent design — methodology that applies whether you're building on a code-first framework, an agent SDK, or a low-code builder.

**Duration:** ~2 hours across 5 lessons plus framework methodology reading
**Prerequisites:** Software engineering fundamentals and basic familiarity with APIs.
**What you'll produce:** A complete scope document and design, ready to implement on whichever agent framework your team uses.

!!! note "Design, build, and capstone lessons have moved to the framework"
    Earlier versions of this pathway had framework-specific lessons on design (Lesson 5), build (Lesson 6), and a capstone hands-on coding exercise (Lesson 8). Those lessons were removed when the framework was decoupled from specific platforms — they were teaching one specific framework's mechanics, and the framework itself should not promote any particular platform. The equivalent content now lives in the platform-neutral methodology pages for [Stage 4: Design](../stages/04-design.md) and [Stage 5: Build](../stages/05-build.md), and in the platform-neutral worked examples ([CSM](../worked-example/design.md), [BA](../worked-example-ba/design.md), [SE](../worked-example-se/design.md)). Read those in place of the removed lessons, then use your chosen platform's documentation to translate the design into an implementation.

## Recommended pace

| Option | Schedule |
|---|---|
| **Intensive** (2–3 hours) | All 5 lessons plus Stage 4 and Stage 5 methodology reading in one sitting |
| **Spread out** (1 week) | Lessons 1–3 on Day 1, Lesson 4 on Day 2, methodology reading on Day 3, Lesson 7 on Day 4 |

!!! tip "Start with Lesson 1"
    Even if you already work in an agent framework, Lesson 1 explains how the framework's stages map to the engineering decisions you'll make. Understanding this mapping makes the methodology reading significantly more productive.
